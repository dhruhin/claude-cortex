# Tags

Global tag registry for the vault. Check here before creating new tags.

## Format

Use hierarchical tags: `#category/subcategory`

---

## Categories

### Meta
- `#meta` - About the vault itself

### Status
- `#status/active` - Currently in progress
- `#status/archived` - Completed or no longer relevant
- `#status/review` - Needs review

### Content Type
- `#journal` - Daily notes
- `#daily` - Daily note marker
- `#meeting` - Meeting notes
- `#project` - Project-related
- `#idea` - Ideas and thoughts
- `#reference` - Reference material

### Note Type (Zettelkasten)
- `#fleeting` - Quick capture, to be processed
- `#literature` - Notes from reading/research
- `#permanent` - Refined, standalone notes

### Areas
- `#area/career`
- `#area/health`
- `#area/finances`
- `#area/relationships`

### Projects
- `#project/project-name` - Tag per active project

### Topics
- `#topic/technology`
- `#topic/productivity`
- `#topic/learning`
- (Add as needed)

---

## Usage Guidelines

1. **Check before creating**: Search this file before making new tags
2. **Keep it flat-ish**: Max 2 levels deep (`#category/subcategory`)
3. **Use lowercase**: Consistency in naming
4. **Be specific**: `#project/auth-migration` not just `#project`
